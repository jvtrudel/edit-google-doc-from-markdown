# Besoins utilisateurs

Ce dossier contient le recensement des besoins des utilisateurs du système.

## Rôle

Un **besoin** exprime un problème ou une attente de l'utilisateur, formulé du point de vue de l'utilisateur. Il ne décrit pas la solution, mais le « pourquoi ».

## Convention de nommage

`NNN-description-courte.md` (ex: `001-editer-google-doc.md`)

## Format

Voir `_template.md` pour le format attendu.

## Liens

- Un besoin peut être couvert par un ou plusieurs **requis** (`.dev/requirements/`)
- Un besoin peut être satisfait par une ou plusieurs **fonctionnalités** (`.dev/features/`)
