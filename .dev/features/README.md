# Fonctionnalités

Ce dossier décrit les capacités fournies à l'utilisateur par le système.

## Rôle

Une **fonctionnalité** est une capacité visible du système, orientée utilisateur. Elle regroupe un ensemble cohérent de requis qui produit une valeur métier.

## Convention de nommage

`NNN-description-courte.md` (ex: `001-edition-google-doc.md`)

## Format

Voir `_template.md` pour le format attendu.

## Liens

- Une fonctionnalité satisfait un ou plusieurs **besoins** (`.dev/needs/`)
- Une fonctionnalité réalise un ou plusieurs **requis** (`.dev/requirements/`)
- Une fonctionnalité peut être détaillée dans une **spécification** (`.dev/specs/`)
