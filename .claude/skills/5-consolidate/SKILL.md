- Lire le ticket et le plan de la ronde courante

- Obligatoirement :
  1. Déterminer le numéro de la prochaine version (semantic versioning : vMAJEUR.MINEUR.PATCH)
  2. Produire un changelog listant les changements depuis la dernière version
  3. Produire un release note résumant les changements pour l'utilisateur
- Écrire ces informations dans le ticket ou dans un fichier dédié
- Demander validation à l'utilisateur
